# UI/UX Design Brief

* [cite_start]**Design Paradigm:** Glassmorphism and Minimalist[cite: 42].
* [cite_start]**Color Palette:** Deep midnight blues and slate greys (dark mode default) with vibrant, glowing accent colors (e.g., neon cyan or electric purple) for primary actions and verified ZK proofs[cite: 43].
* [cite_start]**Visual Hierarchy:** Clean, card-based layouts heavily featuring frosted glass effects (semi-transparent backgrounds with backdrop blur)[cite: 44].
* [cite_start]**User Feedback:** Because ZK proof generation via the local prover takes processing time, the UI must include sleek, technical loading animations indicating "Generating Zero-Knowledge Proof..." or "Verifying Local Witness..." to keep the user engaged[cite: 45].
* [cite_start]**Data Visualization:** Use minimal, abstract visual indicators for credit health rather than raw numbers, reinforcing the privacy-first nature of the application[cite: 46].
